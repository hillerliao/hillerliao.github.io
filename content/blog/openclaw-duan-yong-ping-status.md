# 如何用 OpenClaw 智能监控段永平动态：一个“只看重点”的投资信息自动化实战


过完年相信你已经听说了OpenClaw这个词。简单说，它就是一个更厉害的类似Siri+DeepSeek一样的智能助理。假如你已经拥有了它，下一步可能想知道可以用来干什么？今天这篇文章我将用一个实操案例，分享我的使用体验，欢迎拍砖。

如果你是某个投资达人，比如段永平的关注者，你可能想跟踪他的的最新动态。（如果关注娱乐明星，道理也一样）。但有什么方式能够比较便捷又高效呢？

以投资领域的段永平为例，他平时只在雪球发言，雪球提供了用户动态订阅功能，这本身是个有用的基础能力——你可以订阅感兴趣的投资者，第一时间收到更新。

但在实际使用中，你会发现：所有动态都会推送过来，投资的、生活的、闲聊的，**混在一起**。

有办法能只看到他关于投资的关键发言吗？

有。我最近就尝试用 OpenClaw 做了一个自动化监控系统，全程没手动敲一行代码，就把事情办了。

## 实际效果预览

这是我在 Telegram （类似微信的一款软件）收到的推送：

```
🚨 段永平雪球更新【重点】
📝 内容：回复网友关于期权操作和阿里持仓看法
💡 看法：70元买阿里足够便宜，体现价值投资理念
```

“【重点】”是我设置的条件告警——段永平提到任何具体公司时（比如苹果、腾讯、阿里、茅台、Google 等），系统需要帮我自动标记并给出 AI 分析。


## 总体架构

```
雪球 → RSSHub → OpenClaw定时 → 
AI处理 → 消息推送（Telegram、微信等）
```

我的OpenClaw是部署在云服务器上的，便宜又省事，24小时运转。如果你想用闲置的旧手机，也能部署，可参见《[不用买服务器！旧手机免费运行 OpenClaw](https://mp.weixin.qq.com/s/KASWkkuD2-Q5f22FDQ7bNA)》


## 实现过程

全程自然语言对话，我跟 OpenClaw 聊了几轮，系统就搭建好了。下面是大致过程。

### 1. 说出总体目标

```
我：帮我监控投资人段永平的最新动态。
可以通过rss地址来获取新内容。
```

并给了它一个RSS地址 `https://<my domain name>/xueqiu/user/1247347556`。该地址是借助开源RSS生成工具`RSSHub-Python`生成的。RSS就是一种通用的数据格式，将段永平的帖子按照机器友好的数据格式来展示。

印象中还跟OpenClaw强调了必须是新帖子时才提醒。我给OpenClaw用的是价格友好的MiniMax模型，偶尔需要多叮嘱几句，怕OpenClaw想不全面。

### 2. 部署 RSSHub

不过一开始给的RSS地址不稳定，于是：

```
我：能部署这个吗？
https://github.com/hillerliao/rsshub-python

AI：有 Docker/Vercel/本地三种方式

我：docker 安装吧

AI：遇到网络问题...

我：那就本地部署

AI：改用 pip + git clone，成功
```

中间遇到网络问题，换本地部署方案就解决了。OpenClaw 全程自己处理，我没说几句。要是在以前，部署这个项目估计就要折腾很久。

插一句，运行这个项目对机器配置有一定要求。如果是个人电脑一般没问题，如果是云服务器，最好是4G内存以上。

此外，如果服务器在中国大陆境内，没用这个[链接](https://gitee.com/hillerliao/rsshub-python)。或者告诉OpenClaw用 `r.jina.ai`去访问无法访问的链接。

### 3. 内容过滤

```
我：提到具体公司时要重点提醒

AI：好的……
```

OpenClaw 会用 AI 来判断——不是简单的关键词匹配，而是真正理解文章内容。例如段永平说"今天天气很好，吃了个苹果"，AI 不会标记为投资相关；但如果说"70元买入阿里很便宜"，就会标记为重点。这就是传统软件不太容易搞定的地方。

### 4. 消息推送通道

```
我：推送到 Telegram

AI：配置完成
```

## 用其他工具不也能实现吗？

之前我用 N8N 做过类似的工作流，优点是可视化，但缺点也很明显。

|      | N8N     | OpenClaw |
| ---- | ------- | -------- |
| 部署   | 需要部署该项目  | 没有部署的烦恼，自然语言，几句话搞定    |
| 配置   | 画节点、写代码，反复调试 | 自然语言     |
| AI能力 | 较弱，引入AI节点能实现       | 内置，天生就是AI基因       |

当时费了一些功夫让N8N判断是否为新帖子。因为需要借助代码实现，而自己又不太会JS，所以就很头疼。

核心差异：OpenClaw 可以理解内容——它知道"提到苹果腾讯阿里要重点提醒"，而 N8N 只能做简单的关键词匹配。

我用了 MiniMax 的模型，它的中文理解能力比较好，能准确区分：
- "今天吃了个苹果"（生活）→ 不标记
- "苹果护城河很深"（投资）→ 重点提醒

这是模型差异带来的，不是工具好坏的问题。
