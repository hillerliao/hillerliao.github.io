# 段永平发言太杂？我用 OpenClaw 自动筛选有价值观点（附实操）

简单一句话概括：

> 它是一个**能理解你意图、替你干活的 AI Agent**，有点像 *Siri + DeepSeek + 自动化工具* 的结合体。

很多人拿到 OpenClaw 后，第一反应往往是：

> 👉 *“这东西到底能干啥？”*

这篇文章，我不讲概念，只讲一个**我自己已经跑起来的真实案例**：
**如何用 OpenClaw，智能监控段永平的投资动态，只推送“有价值的信息”。**

如果你关注的不是投资人，而是娱乐明星、行业大佬、KOL，道理完全一样。

## 一个痛点：信息太多，但重点太少

如果你是价值投资者，大概率关注过 **段永平**。

他的特点很鲜明：

* 几乎**只在雪球发言**
* 发言频率不高，但**含金量极高**
* 内容混杂：

  * 投资观点
  * 生活闲聊
  * 回复网友

雪球（雪球）本身提供了「订阅用户动态」功能，这已经是一个不错的基础能力。

但实际用下来会发现一个问题：

> ❌ **所有内容都会推送，混在一起**

你真正想看到的，可能只有这些：

* 提到 **具体公司**
* 涉及 **买入 / 持仓 / 估值判断**
* 能反映其**投资逻辑变化**的发言

那么问题来了：

> **能不能：只在“有投资价值”的时候提醒我？**

答案是：**可以，而且我已经跑起来了。**


## 实际效果预览

这是我在 **Telegram** 上收到的真实推送示例：

```
🚨 段永平雪球更新【重点】
📝 内容：回复网友关于期权操作和阿里持仓看法
💡 看法：70元买阿里足够便宜，体现其长期价值投资理念
🔗 原文：https://xueqiu.com/1247347556/376663072
```

其中这个 **「【重点】」**，并不是我手动加的，而是我给 OpenClaw 设定的**条件告警规则**：

> **只要段永平的发言中：**
>
> * 提到具体公司（如：苹果、腾讯、阿里、茅台、Google 等）
> * 且语义上与投资相关
>   👉 就自动标记为重点，并给出 AI 解读

## 实现过程：全程自然语言，没写代码

这是我最想分享的部分。

整个系统，我**没有手动写一行代码**，全部是跟 OpenClaw 对话完成的。

### ① 先说清楚目标

```
我：帮我监控投资人段永平的最新动态，
    可以通过 RSS 地址获取新内容。
```

然后我把 RSS 地址丢给它：

```
https://<my domain>/xueqiu/user/1247347556
```

这个 RSS 是我用开源项目 **RSSHub-Python** 生成的。
本质上，它把段永平的雪球帖子，转换成**机器更容易处理的数据格式**。

我还特意强调了一点：

> ⚠️ **只在“有新帖子”时才提醒**

---

### ② RSS 不稳定？那就让 OpenClaw 自己部署

一开始我用的是别人提供的 RSS 地址，不太稳定，我就想到自己部署一套RSSHub-Python服务。于是对话变成了这样：

```
我：这个项目能部署吗？
   https://github.com/hillerliao/rsshub-python

AI：支持 Docker / Vercel / 本地部署

我：Docker 吧

AI：网络问题……

我：那就本地部署

(若干分钟后...)

AI：部署成功，地址 http://127.0.0.1/xueqiu/user/1247347556
```

为了测试验证，可以让OpenClaw给你请求该地址实际获取到的内容。

如果你获取到的是这种字样的： `Playwright not supported on Vercel`，发给OpenClaw，让它安装一下 Playwright，具体是啥不用管。

整个过程，**OpenClaw 自己解决依赖、环境、启动问题**，我基本只是做选择题。

一个小插曲，它把我已经存在的一个网页服务给搞成404无法访问了，幸好后来让它帮忙修复了。改坏东西是OpenClaw容易犯的毛病。

> 如果你是云服务器，建议 4G 内存以上  
> 中国大陆服务器可用 [Gitee 镜像](https://gitee.com/hillerliao/rsshub-python)，或让 OpenClaw 用 `r.jina.ai` 访问 GitHub 版地址。

### ③ 核心价值：不是关键词，而是“理解内容”

我只说了一句话：

```
我：提到具体公司时要重点提醒
```

OpenClaw 背后做的事情，其实是：

* **语义理解**
* **上下文判断**
* **投资 vs 生活区分**

举个最关键的例子：

* ✅「70 元买阿里很便宜」 → **重点**
* ❌「今天吃了个苹果」 → **不提醒**

这一步，是传统自动化工具**最难做到的地方**。

## 那用别的工具不行吗？

行，但体验完全不一样。

我之前用 **N8N** （一个开源的低代码服务）搭过类似流程，简单对比一下：

| 对比项   | N8N        | OpenClaw |
| ----- | ---------- | -------- |
| 部署    | 需要自己部署     | 几乎无感     |
| 配置    | 画节点 + 写 JS | 自然语言     |
| 判断逻辑  | 关键词为主      | 语义理解     |
| AI 能力 | 外挂         | 内置核心     |

当时为了判断“是否为新帖子”，我写了不少 JS，调试非常痛苦。

**本质差异只有一句话：**

> OpenClaw 能“看懂你在说什么”，
> 而传统自动化工具，只是在“对字符串”。

我用的是 MiniMax 模型，它在中文语境下，对投资语义的区分非常准确。

而且后续要优化，直接和机器人通过聊天沟通就行了，不需要登录相关工具后台去来回调试操作。
