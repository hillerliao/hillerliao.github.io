> OpenClaw 或 PicoClaw 这类"虾产品"安装难、配置难、维护更难。但现在，你只需要动动嘴，剩下的交给 Hermes Agent 就可以了。

## 一、想试那只"虾"，第一步就卡住

OpenClaw 用着用着就碰到各种各样的问题。你听说一个叫做 PicoClaw 的类 OpenClaw 产品，它号称轻量、省钱、性能猛——"只要 10 美元硬件，内存 <10MB，启动 <1s"。

这不就是 OpenClaw 绝佳的替代品吗？心动不如行动，打开 GitHub 瞧瞧。

- 下载哪个版本？Linux x86_64 还是 arm64？
- 下载完怎么启动？`gateway` 是啥？`agent` 又是啥？
- 配置文件里 `allow_from` 是什么？不填能用吗？
- 升级的时候...等等，我连当前版本是啥都不知道！

更崩溃的是：

- 命令行黑黑的窗口，输错一个字符就报错
- 文档全是英文，还有技术黑话（hot_reload、sandbox、spawn...）
- 一修改配置，服务起不来了，不知道哪错了
- 想升级，怕把配置搞丢，怕升级后连不上飞书
- 升级失败？回滚是啥？备份了哪些文件？完全懵住

**一句话**：

> **"虾"是好虾，但小白根本不会煮** —— 不是不想学，是怕搞坏、怕背锅、怕花时间学完一次再也不用。

| 操作 | 你要干的 | Hermes 帮你干的 |

| **安全** | 1. 想想要不要备份 2. 手动 cp 文件，记备份路径 3. 升级失败时要想得起备份在哪 | 自动备份到带时间戳的目录，最后附回滚命令 |

## 五、Hermes Agent 运维模式的三大安全保障

### 1️⃣ 永远有后悔药

> 升级前自动备份到 `~/picoclaw-backup-20260421-095056`
> 回滚命令它主动告诉你，不用你费劲找备份在哪。

### 2️⃣ 它只干官方推荐的事

> 不会用奇怪参数，而是找到 `upgrade_picoclaw.sh` 官方脚本，走稳定流程。
> 你不需要判断哪个分支靠谱，它帮你选 stable。

### 3️⃣ 事后能审计

> 聊天记录就是完整操作日志。
> 哪天服务出问题，翻翻聊天就知道：那天升级到哪个版本、改了哪些配置、备份在哪里。
